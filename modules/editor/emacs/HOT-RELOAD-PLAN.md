# Emacs + Nix: Hot-Reload Configuration Plan

## Current Architecture

The system works in two layers:

### Layer 1: `emacs-init.nix` — Declarative Emacs Module Generator

This file defines a Nix module (`programs.emacs.init`) that lets you declare
Emacs packages via the `usePackage` option — a submodule that mirrors
`use-package` keywords:

| Nix option       | `use-package` keyword | Type                           |
|------------------|-----------------------|--------------------------------|
| `enable`         | —                     | `bool`                         |
| `package`        | `:ensure`             | `str` or `packageFunctionType` |
| `after`          | `:after`              | `listOf str`                   |
| `bind`           | `:bind`               | `attrsOf str`                  |
| `bindLocal`      | `:bind :map`          | `attrsOf (attrsOf str)`       |
| `bindKeyMap`     | `:bind-keymap`        | `attrsOf str`                  |
| `chords`         | `:chords`             | `attrsOf str`                  |
| `custom`         | `:custom`             | `lines`                        |
| `customFace`     | `:custom-face`        | `attrsOf str`                  |
| `command`         | `:commands`           | `listOf str`                   |
| `defines`        | `:defines`            | `listOf str`                   |
| `demand`         | `:demand`             | `bool`                         |
| `defer`          | `:defer`              | `bool` or `ints.positive`      |
| `diminish`       | `:diminish`           | `listOf str`                   |
| `functions`      | `:functions`          | `listOf str`                   |
| `hook`           | `:hook`               | `listOf str`                   |
| `mode`           | `:mode`               | `listOf str`                   |
| `noRequire`      | `:no-require`         | `bool`                         |
| `preface`        | `:preface`            | `lines`                        |
| `init`           | `:init`               | `lines`                        |
| **`config`**     | **`:config`**         | **`lines` or `path`** *(recently changed)* |
| `extraConfig`   | remaining             | `lines`                        |
| `earlyInit`     | early-init.el         | `lines`                        |

The `assembly` option (read-only, internal) gathers all of the above and
generates a `(use-package ...)` sexp. All assemblies are concatenated into
`initFile`, which becomes `hm-init.el` — built as an Emacs package via
`trivialBuild` and installed to the Nix store.

### Layer 2: `default.nix` — The User's Config

This is a ~3500-line file that wires everything together. It:
- Defines `programs.emacs.init.usePackage` entries for every package
- Inline Elisp in `config`, `init`, `preface`, `custom`, `extraConfig` fields
- References Nix store paths for external binaries (e.g. `${pkgs.ripgrep}/bin/rg`)
- Sets `earlyInit` and `prelude`/`postlude` blocks
- Adds `home.packages` (system tools Emacs needs)
- Creates `home.file` entries (snippets via `mkOutOfStoreSymlink`, etc.)

### The Final Output

```
~/.emacs.d/early-init.el  →  (require 'hm-early-init)
~/.emacs.d/init.el        →  (require 'hm-init)
```

Both `hm-early-init` and `hm-early-init` are built as packages via
`trivialBuild`, with their `.el` source files generated by Nix string
interpolation — including all the inline Elisp from every `usePackage` entry.

**Problem**: Any change to *any* Elisp string requires `home-manager switch`
(Nix rebuild) to take effect. This can take 30-60+ seconds.

## Strategy: Split Static vs Dynamic

The key insight is that not all parts of the config change at the same rate.

| Change frequency | Example                           | Rebuild needed? |
|------------------|-----------------------------------|-----------------|
| Rare             | New package, keybinding change    | **Yes** — structural |
| Frequent         | Tweak a `setq`, adjust a face     | **No** — content   |

We exploit this by:

1. **Keeping structural declarations in Nix** (packages, hooks, keybinds, `:after`, `:mode`, etc.)
2. **Moving frequently-changed Elisp to external files** that Emacs loads at runtime
3. **Providing a reload mechanism** so changes are picked up without restarting Emacs

## Phase 1: Path-Reloading Infrastructure (Foundation)

### 1.1 Fix the `config` path handling

The current change we made:

```nix
config = mkOption {
  type = types.either types.lines types.path;
  ...
};
```

And in the assembly:

```nix
(if builtins.isPath config.config
    then builtins.readFile config.config    # ← embeds content at build time
    else config.config)
```

**Problem**: `builtins.readFile` copies the file content into the Nix store at
build time. Changes to the source file won't be reflected until rebuild.

**Fix**: When a path is given, emit a `(load ...)` form instead:

```nix
(if builtins.isPath config.config
    then "(load \"${config.config}\" no-error)"
    else config.config)
```

This tells Emacs to load the file *at runtime*. But there's a subtlety: the
file must exist on disk when Emacs starts. We have two options:

#### Option A: Absolute path (recommended for hot-reload)

```nix
# In assembly generation:
(if builtins.isPath config.config
    then "(load \"" (toString config.config) "\")"
    else config.config)
```

This works if the path is a stable absolute path outside `/nix/store`
(e.g., `~/dotfiles/emacs/configs/xxx.el`). For `/nix/store` paths,
the file is immutable anyway, so `(load ...)` and `readFile` are equivalent.

#### Option B: Store path with `load`

If the path *is* in `/nix/store`, `load` works fine — but the content is
frozen. This doesn't help hot-reload. So **Option A is the way to go.**

### 1.2 Extend to other `lines` fields

Apply the same `types.either types.lines types.path` pattern to:

- `init` — `:init` block
- `preface` — `:preface` block
- `extraConfig` — trailing config
- `earlyInit` — early-init content
- `prelude` / `postlude` — top-level init.el sections
- `custom` — `:custom` section

Each would get the same treatment:
```nix
(if builtins.isPath config.init
    then "(load \"" (toString config.init) "\")"
    else config.init)
```

### 1.3 Add `mkOutOfStoreElisp` helper

Create a helper that symlinks an external `.el` file and returns the load
path, so users don't have to think about absolute paths:

```nix
# In emacs-init.nix or a separate lib file:
mkOutOfStoreElisp = hm: relPath:
  let
    src = hm.config.lib.file.mkOutOfStoreSymlink
            "${hm.config.home.homeDirectory}/${relPath}";
    # The symlink target is the actual path Emacs should load
  in
    src;
```

Usage in `default.nix`:
```nix
usePackage.corfu.config = ./configs/corfu.el;  # ← path, auto-tracked
# or, for truly out-of-store:
usePackage.corfu.config = mkOutOfStoreElips hm "dotfiles/emacs/configs/corfu.el";
```

## Phase 2: Hot-Reload Mechanism in Emacs

### 2.1 Add a "personal config" file

Add a well-known location for ad-hoc Elisp that's always loaded:

```nix
# In emacs-init.nix, add to initFile:
initFile = ''
  ...
  ;; Load personal config if it exists — no rebuild needed
  (load "${config.my.homeDirectory}/.emacs.d/personal.el" t)
'';
```

Users can edit `~/.emacs.d/personal.el` freely and reload with `M-x load-file`.

### 2.2 Add per-package external config files

For each `usePackage` entry with a path-typed `config`, the generated assembly
will include `(load "/absolute/path/to/foo-config.el")`. Users edit those
files and reload.

### 2.3 Auto-reload with `auto-revert-mode`

The config already enables `auto-revert-mode` globally. Combined with
`load` statements, Emacs can detect when an external config file changes.

Add a custom minor mode that watches loaded config files:

```elisp
;;; hm-hot-reload.el --- Reload external config on change
(defvar hm-external-config-files nil
  "List of external config files loaded via hm-init.")

(defvar hm-hot-reload-timer nil
  "Timer for polling external config file changes.")

(defun hm-reload-external-configs ()
  "Reload all external config files that have changed."
  (interactive)
  (dolist (file hm-external-config-files)
    (when (and (file-exists-p file)
               (file-newer-than-file-p file (make-temp-file "hm-reload")))
      (load file t))))

(define-minor-mode hm-hot-reload-mode
  "Reload external Emacs config files when they change."
  :global t
  :lighter " 🔥"
  (if hm-hot-reload-mode
      (setq hm-hot-reload-timer
            (run-with-idle-timer 2 t #'hm-reload-external-configs))
    (cancel-timer hm-hot-reload-timer)))
```

**Better approach** — use `file-notify` (inotify/kqueue-based, no polling):

```elisp
(defvar hm--file-watchers nil)

(defun hm-watch-config-file (file)
  "Watch FILE for changes and reload it."
  (push (file-notify-add-watch file '(change)
          (lambda (_event)
            (when (file-exists-p file)
              (load file t))))
        hm--file-watchers))
```

### 2.4 Add `hm-reload` interactive command

```elisp
(defun hm/reload-config ()
  "Reload all external config files."
  (interactive)
  (dolist (file hm-external-config-files)
    (when (file-exists-p file)
      (load file t)))
  (message "Reloaded %d config files" (length hm-external-config-files)))
```

## Phase 3: Directory Structure

Create a canonical directory for external Elisp configs:

```
~/.emacs.d/
├── init.el                  # ← Nix-generated (require 'hm-init)
├── early-init.el            # ← Nix-generated
├── personal.el              # ← Hot-reloadable ad-hoc config (Phase 2.1)
├── external/                # ← Hot-reloadable per-package configs
│   ├── corfu-config.el
│   ├── meow-config.el
│   ├── org-config.el
│   ├── vertico-config.el
│   └── ...
└── snippets/                # ← Already symlinked via mkOutOfStoreSymlink
```

In `default.nix`:
```nix
home.file = {
  ".emacs.d/external".source =
    hm.config.lib.file.mkOutOfStoreSymlink
      "${config.my.homeDirectory}/${config.my.workspaceDirectory}/nix-home/modules/editor/emacs/configs";
};
```

Then reference these files:
```nix
usePackage.corfu.config = ./configs/corfu-config.el;  # Nix path → store path
# or for true hot-reload (outside /nix/store):
usePackage.corfu.config = let
  external = "${config.my.homeDirectory}/${config.my.workspaceDirectory}/nix-home/modules/editor/emacs/configs";
in external + "/corfu-config.el";  # ← absolute path, loaded at runtime
```

## Phase 4: Implementation Steps

### Step 1: Modify `emacs-init.nix` — Path-aware assembly generation

For each text-generating option that supports `types.path`, change the assembly
to emit `(load ...)` instead of embedding content:

```nix
# Helper function (add near top of the let block):
mkElispOrLoad = name: content:
  if builtins.isPath content
  then "(load \"${toString content}\" t)"
  else content;
```

Then use it in the assembly:
```nix
++ optionals (config.config != "" && config.config != null) [
  ":config"
  (mkElispOrLoad "config" config.config)
]
++ optionals (config.init != "") [
  ":init"
  (mkElispOrLoad "init" config.init)
]
++ optionals (config.preface != "") [
  ":preface"
  (mkElispOrLoad "preface" config.preface)
]
```

### Step 2: Update option types

For each option that should support hot-reload:
```nix
config = mkOption {
  type = types.either types.lines types.path;
  default = "";
  ...
};
init = mkOption {
  type = types.either types.lines types.path;
  default = "";
  ...
};
preface = mkOption {
  type = types.either types.lines types.path;
  default = "";
  ...
};
extraConfig = mkOption {
  type = types.either types.lines types.path;
  default = "";
  ...
};
```

### Step 3: Add the `personal.el` autoload

In `emacs-init.nix`, append to `initFile`:
```nix
initFile = ''
  ...
  ;; Load user's hot-reloadable personal config
  (let ((personal-file (expand-file-name "personal.el" user-emacs-directory)))
    (when (file-exists-p personal-file)
      (load personal-file t)))

  (provide 'hm-init)
  ;; hm-init.el ends here
'';
```

### Step 4: Migrate large config blocks to external files

Start with the largest/most-frequently-changed packages. For example,
`meow` has ~100 lines of keybind config:

```nix
# Before (in default.nix):
meow = {
  enable = true;
  config = ''
    (defun meow-setup () ...)
    (meow-setup)
    (meow-global-mode 1)
  '';
};

# After:
meow = {
  enable = true;
  config = ./configs/meow.el;  # ← loaded at runtime, editable on the fly
};
```

Create `modules/editor/emacs/configs/meow.el`:
```elisp
(defun meow-setup ()
  (setq meow-cheatsheet-layout meow-cheatsheet-layout-qwerty)
  ;; ... all the keybinds ...
  )
(meow-setup)
(meow-global-mode 1)
```

### Step 5: Add `hm-reload` to early-init or prelude

```nix
# In default.nix prelude:
prelude = ''
  ...
  (defvar hm-external-config-files nil
    "External config files loaded by hm-init.")

  (defun hm/reload-config ()
    "Reload all external config files."
    (interactive)
    (dolist (file hm-external-config-files)
      (when (file-exists-p file)
        (load file t)))
    (message "Reloaded %d config files" (length hm-external-config-files)))
'';
```

And modify the assembly generation to track loaded files:
```nix
# In mkElispOrLoad, also register the file:
mkElispOrLoad = name: content:
  if builtins.isPath content
  then "(let ((f \"${toString content}\")) (add-to-list 'hm-external-config-files f) (load f t))"
  else content;
```

### Step 6: Add file-notify-based auto-reload (optional, advanced)

```nix
# In prelude:
prelude = ''
  ...
  (defvar hm--file-watchers nil
    "File-notify watch descriptors for hot-reload.")

  (defun hm--watch-config-file (file)
    "Watch FILE for changes and reload it."
    (when (and (file-exists-p file) (fboundp 'file-notify-add-watch))
      (push (file-notify-add-watch
              file '(change)
              (lambda (_event)
                (when (file-exists-p file)
                  (load file t)
                  (message "Reloaded: %s" file))))
            hm--file-watchers)))

  (define-minor-mode hm-hot-reload-mode
    "Auto-reload external config files on save."
    :global t
    (if hm-hot-reload-mode
        (dolist (f hm-external-config-files)
          (hm--watch-config-file f))
      (dolist (w hm--file-watchers)
        (file-notify-rm-watch w))
      (setq hm--file-watchers nil)))
'';
```

## What Must Still Rebuild

| Change Type                    | Requires Rebuild? | Why                                   |
|-------------------------------|--------------------|---------------------------------------|
| New Emacs package              | ✅ Yes             | Package must be installed             |
| Remove Emacs package           | ✅ Yes             | Package must be uninstalled           |
| Keybinding changes (`:bind`)   | ✅ Yes             | Part of use-package assembly          |
| `:hook` / `:mode` changes     | ✅ Yes             | Structural, use-package macro          |
| `:after` / `:demand` changes   | ✅ Yes             | Affects load timing                   |
| `setq` / `defvar` in `:config`| ❌ No (if path)    | Loaded at runtime from external file  |
| `:custom` variables            | ❌ No (if path)    | Loaded at runtime                     |
| Any large Elisp block          | ❌ No (if path)    | Loaded at runtime                     |
| `:preface` code                | ❌ No (if path)    | Loaded at runtime                     |
| `earlyInit` code              | ❌ No (if path)    | Loaded at runtime (but requires restart) |
| `prelude` / `postlude`         | ❌ No (if path)    | Loaded at runtime                     |

**Note**: `earlyInit` changes typically require an Emacs restart regardless,
since `early-init.el` runs before the frame is created.

## Recommended Workflow

```
1. Add new package → Nix rebuild (unavoidable)
2. Set up package structure (hooks, modes, after) → Nix rebuild
3. Write config blocks in external .el files → No rebuild needed!
4. Edit .el files, save → auto-reload or M-x hm/reload-config
5. Tweak, iterate, experiment → No rebuild, instant feedback
```

### Gradual Migration Strategy

Don't migrate everything at once. Start with:

1. **Phase 1**: Implement the path-aware infrastructure in `emacs-init.nix`
2. **Phase 2**: Add `personal.el` autoload — immediate benefit, zero migration
3. **Phase 3**: Move your top 5 most-edited packages' `config` fields to `./configs/*.el`
4. **Phase 4**: Move `init` and `preface` for packages you tweak often
5. **Phase 5**: Add `hm-hot-reload-mode` with file-notify

This way you get instant hot-reload benefits incrementally while keeping
structural declarations safely in Nix.
