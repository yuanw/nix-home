#+title: NVIDIA DGX Spark (Project DIGITS) — NixOS

* Overview

This host configuration targets the [[https://www.nvidia.com/en-us/dgx/dgx-spark/][NVIDIA DGX Spark]] (also sold as the Asus Ascent GX10) — a Grace Blackwell-based AI workstation. It uses:

- [[https://github.com/graham33/nixos-dgx-spark][nixos-dgx-spark]] for hardware support (NVIDIA kernel, GPU drivers, DGX Dashboard)
- [[https://github.com/nix-community/disko][disko]] for declarative disk partitioning
- [[https://github.com/yuanw/ds4][ds4]] — DS4 DeepSeek V4 Flash inference server

* Disk Layout

#+begin_src
/dev/nvme0n1
  ├── ESP (1G)      → /boot   (vfat, UEFI)
  └── root (rest)   → /       (ext4, all state persistent)
#+end_src

Standard ext4 root filesystem — all state survives reboots normally. No tmpfs,
no special persistence layer needed.

* Networking

The DGX Spark is a headless server, so we use =systemd-networkd= via
=networking.useDHCP = true= instead of NetworkManager. This provides
predictable, declarative wired networking without requiring a desktop session.

All Ethernet interfaces get DHCP automatically.

mDNS (Avahi) is enabled, so the host is discoverable as =dgx-spark.local= on
the LAN.

* SSH Access

** SSH host keys

SSH host keys are stored on the ext4 root filesystem like any normal NixOS
system — they persist across reboots automatically.

** Connecting

The =yuanw= user has password login disabled — authentication is SSH key only
(=PasswordAuthentication = false=). Your public key is pre-configured:

#+begin_src
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIMSvr2qkdnG03/pGLo3aCFTnwmvojKO6m/W74ckC1RPW me@yuanwang.ca
#+end_src

#+begin_src shell
ssh yuanw@dgx-spark
# or by IP:
ssh yuanw@<ip-address>
#+end_src

** Finding the IP address

If you don't know the DGX Spark's IP:

- Use =dgx-spark.local= (mDNS/Avahi): =ping dgx-spark.local=
- Check your router's DHCP client list
- Use =arp-scan= on the same subnet
- Connect a monitor and keyboard for the first boot to get the IP from the console

** Setting a password (optional)

The account starts with a locked password (SSH keys only). To add a password
for console login:

#+begin_src shell
# On the DGX Spark console or via SSH:
passwd yuanw
#+end_src

To generate a hashed password for the config instead:

#+begin_src shell
mkpasswd -m sha-512
#+end_src

Then set =users.users.yuanw.hashedPassword= in =configuration.nix=.

** Troubleshooting SSH

| Symptom                          | Cause                                    | Fix                                              |
|----------------------------------+------------------------------------------+--------------------------------------------------|
| =Connection refused=             | =sshd= not running or firewall blocking  | Check =systemctl status sshd=, verify port 22     |
| =Host key changed= warning       | Keys regenerated (shouldn't happen)      | Remove old key: =ssh-keygen -R dgx-spark=         |
| =Permission denied (publickey)=  | Key not in authorized_keys               | Verify =~/.ssh/authorized_keys= on the host       |

* DGX Spark Hardware Module

The =hardware.dgx-spark= module (from =nixos-dgx-spark=) enables:

- NVIDIA's custom DGX Spark kernel (6.17+nvidia patches, built from source)
- NVIDIA open GPU driver (=hardware.nvidia.open = true=)
- Podman with NVIDIA container toolkit
- DGX Dashboard at =http://localhost:11000= (proxied from internal port 11001)
- Flox binary cache for pre-built CUDA packages
- Security hardening (=module_blacklist= for vulnerable kernel modules)

* DS4 DeepSeek V4 Flash Inference Server

The [[https://github.com/yuanw/ds4][ds4]] service runs the DeepSeek V4 Flash
inference server as a systemd unit:

- Service: =ds4-server= (systemd, user =ds4=, group =ds4=)
- Default port: 8000 (open in firewall)
- Data directory: =/var/lib/ds4=
- Context size: 100k tokens
- GPU access via =video= group

To interact with the server:

#+begin_src shell
# Check status
systemctl status ds4-server

# View logs
journalctl -u ds4-server -f

# Chat via HTTP API
curl http://dgx-spark:8000/v1/chat/completions \
  -d '{"model":"deepseek-v4","messages":[{"role":"user","content":"Hello"}]}'
#+end_src

* Installation

** Prerequisites

1. Update firmware on the DGX Spark from DGX OS first — NixOS cannot boot from factory firmware.
   See [[https://github.com/graham33/nixos-dgx-spark#firmware-updates][nixos-dgx-spark firmware notes]].
2. Disable Secure Boot in BIOS.

** Option A: USB image

#+begin_src shell
# Build the USB image from the nixos-dgx-spark repo
nix build github:graham33/nixos-dgx-spark#usb-image
sudo dd if=result/iso/*.iso of=/dev/your_usb bs=1M status=progress
#+end_src

Boot from USB, partition with disko, then install.

#+begin_src shell
sudo nix --extra-experimental-features "nix-command flakes" \
  run 'github:nix-community/disko/latest#disko-install' -- \
  --flake .#dgx-spark \
  --disk main /dev/nvme0n1
#+end_src

** Option B: nixos-anywhere (recommended for headless)

#+begin_src shell
nix run github:nix-community/nixos-anywhere -- \
  --flake .#dgx-spark root@<ip>
#+end_src

This will:
1. Partition the NVMe disk using the disko config
2. Install NixOS with all modules
3. Reboot into the new system

To test the disk config in a VM first:
#+begin_src shell
nix run github:nix-community/nixos-anywhere -- \
  --flake .#dgx-spark --vm-test
#+end_src

** Post-install: first boot checklist

- [ ] *Set your user password* (optional, SSH keys already work): =passwd yuanw=
- [ ] *Verify SSH access*: =ssh yuanw@dgx-spark=
- [ ] *Check GPU*: =nvidia-smi= or visit =http://dgx-spark:11000= (DGX Dashboard)
- [ ] *Clone the flake config* (one-time):

#+begin_src shell
ssh yuanw@dgx-spark.local

# One-time setup
sudo mkdir -p /etc/nixos
sudo chown yuanw /etc/nixos
git clone https://github.com/yuanw/nix-home.git /etc/nixos
#+end_src

* Common Operations

** Build the system

#+begin_src shell
nix build .#dgx-spark
#+end_src

** Deploy remotely

#+begin_src shell
nixos-rebuild switch --flake .#dgx-spark --target-host root@dgx-spark
# or by IP:
nixos-rebuild switch --flake .#dgx-spark --target-host root@<ip>
#+end_src

** Factory reset

Since this is a standard ext4 root, a factory reset means reinstalling from
scratch (disko + nixos-anywhere). There is no tmpfs persistence layer to wipe.
